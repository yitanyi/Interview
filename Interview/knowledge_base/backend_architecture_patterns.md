# 后端架构设计与高并发系统实战

## 一、高并发系统通用设计原则
### 1. 分层架构与领域驱动设计
- **COLA 架构**：阿里巴巴复杂业务架构最佳实践
- **防腐层模式**：隔离第三方服务依赖变更

### 2. 流量治理策略
- **限流算法**：滑动窗口 vs 漏桶 vs 令牌桶，Sentinel 集群限流原理
- **熔断降级**：Hystrix 断路器状态机与半开状态探测

## 二、数据一致性架构设计
### 1. 分布式事务终极方案
- **本地消息表 + 定时任务**：实现最大努力通知
- **RocketMQ 事务消息**：确保本地事务与消息发送的原子性

### 2. 最终一致性与对账系统
- **增量对账与全量对账**：夜间 T+1 全量核对与准实时差异补偿

## 三、微服务治理深度解析
### 1. 服务网格
- **Sidecar 模式**：Istio 流量劫持原理（Iptables 与 Envoy）

### 2. 可观测性三支柱
- **Logging**：ELK 日志采集与结构化日志规范
- **Tracing**：Jaeger 全链路追踪，跨线程/跨进程 TraceId 透传方案
- **Metrics**：Prometheus + Grafana 告警规则配置

## 四、安全防护体系
### 1. API 安全
- **JWT 与 OAuth2.0**：Token 无状态续期与黑名单机制
- **幂等性设计**：基于 Redis Token 或数据库唯一索引防重提交

### 2. 攻击防御
- **防刷策略**：布隆过滤器 + 滑动窗口计数
- **SQL 注入与 XSS**：ORM 框架底层防护与富文本白名单过滤

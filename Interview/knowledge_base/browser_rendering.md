# 浏览器渲染原理与性能优化

## 一、浏览器导航与加载流程
当你在地址栏输入URL并回车，发生了什么？
1.  **DNS解析**：将域名解析为IP地址。
2.  **建立TCP连接**：三次握手。
3.  **发送HTTP请求**：请求HTML文档。
4.  **服务器响应**：返回HTML文档流。

## 二、浏览器渲染主流程（关键渲染路径）
浏览器拿到HTML文档后，渲染引擎开始工作，主要包含以下五个步骤：

### 1. 构建DOM树
- **过程**：HTML解析器将HTML文档解析为一系列Token，然后构建成节点（Node），最终形成一棵由父子、兄弟关系组成的树形结构——DOM（文档对象模型）树。
- **特点**：解析过程是逐步进行的，遇到 `<script>` 标签（尤其是没有 `async` 或 `defer` 属性时）会**阻塞DOM构建**，因为JS可能会修改DOM。

### 2. 构建CSSOM树
- **过程**：CSS解析器解析CSS（包括内联、外联、行内样式），构建成一棵CSSOM（CSS对象模型）树。
- **特点**：CSS解析**不会阻塞DOM树的构建**，但**会阻塞渲染**。因为JS脚本在下载和执行时，可能会查询样式信息，所以CSSOM的构建会阻塞JS的执行。

### 3. 构建渲染树（Render Tree）
- **过程**：将DOM树和CSSOM树合并成渲染树。
- **关键**：渲染树只包含**需要显示的节点**（如 `display: none` 的节点不会被包含，但 `visibility: hidden` 的节点仍会占据空间，所以会被包含）。

### 4. 布局（Layout / Reflow）
- **过程**：计算渲染树中每个节点的几何位置（宽、高、位置）。这是一个相对复杂的计算过程，因为涉及相对单位、浮动、定位等。

### 5. 绘制（Paint / Repaint）
- **过程**：将每个节点的像素绘制到屏幕上，包括文字、颜色、阴影、边框等。这一步通常是在多个层（Layer）上分别绘制，然后由合成器（Compositor）合成最终图像。

## 三、重排（Reflow）与重绘（Repaint）

### 1. 重排
- **触发条件**：当操作导致**布局发生变化**时。
  - 添加/删除可见的DOM元素。
  - 元素位置、尺寸、内容改变。
  - 浏览器窗口大小改变（`resize` 事件）。
  - 获取某些特定属性值（如 `offsetTop`、`scrollTop`、`clientTop`、`getComputedStyle()` 等），浏览器为了返回准确值，会立即强制刷新队列，触发重排 [citation:1]。
- **性能影响**：开销巨大。通常会引起子元素甚至祖先元素的重新计算。

### 2. 重绘
- **触发条件**：当操作改变元素的**外观但不影响布局**时（如颜色、背景色、`visibility`）。
- **性能影响**：开销相对较小，但仍需遍历所有元素，消耗GPU资源。

### 3. 优化方案
- **读写分离**：避免在同一个事件循环中先读后写（导致强制重排）。可以使用 `fastdom` 库或手动将读写操作分批。
- **批量修改DOM**：使用 `document.createDocumentFragment()` 创建文档片段，在内存中批量操作完成后，再一次性插入DOM树。
- **脱离文档流**：对需要多次修改的元素，先设置 `display: none`（触发一次重排），修改100次后再设置 `display: block`（触发一次重排）。通过脱离文档流避免回流计算。
- **使用 `transform` 代替 `top/left`**：`transform` 触发的是**合成**，不涉及重排和重绘，由GPU处理，性能最佳。`will-change: transform` 可以提前告诉浏览器开启硬件加速 [citation:1]。
- **防抖/节流**：对 `resize` 和 `scroll` 事件进行防抖/节流处理。

## 四、CSS与JS加载对渲染的影响

### 1. CSS阻塞
- CSS会阻塞渲染（Render Tree构建），但不会阻塞DOM解析。

### 2. JS阻塞
- 普通 `<script>`（无 `async`/`defer`）会**阻塞DOM解析**（因为JS可能会修改DOM）。
- **`async`**：脚本下载与HTML解析并行，下载完成后立即执行（执行时会阻塞解析）。适用于无依赖的独立脚本（如百度统计）。
- **`defer`**：脚本下载与HTML解析并行，但**等待HTML解析完成**（在 `DOMContentLoaded` 事件之前）按顺序执行。适用于需要操作DOM的脚本。
